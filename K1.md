# K-1 Fiancé Visa Quiz

Dedicated assessment tool for K-1 fiancé visa eligibility.

## Overview

The K-1 quiz helps couples determine if they qualify for a fiancé visa, which allows a foreign national engaged to a US citizen to enter the US for marriage.

**Route:** `http://localhost:5173/k1`

## K-1 Visa Basics

| Role | Who | Responsibility |
|------|-----|----------------|
| **Petitioner** | US citizen | Files Form I-129F, meets income requirements |
| **Beneficiary** | Foreign fiancé(e) | Receives visa, enters US, marries within 90 days |

**Key rules:**
- Only US citizens can petition (not green card holders)
- Couple must have met in person within last 2 years (with limited exceptions)
- Must marry within 90 days of beneficiary entering US

## Quiz Flow

The quiz supports **both perspectives** - either the US citizen or foreign fiancé(e) can take it.

### Question 1: Role Selection
```
"Which person in the relationship are you?"
- I am the US citizen
- I am the foreign fiancé(e)
```

This determines:
- Which questions are shown (US citizens skip the citizenship verification question)
- Question wording (e.g., "your income" vs "your fiancé(e)'s income")

### Question Flow

| # | Question | Conditional |
|---|----------|-------------|
| 1 | Role selection | Always shown |
| 2 | Relationship status | Always shown |
| 3 | Partner's citizenship | Foreign nationals only |
| 4 | In-person meeting | If engaged + valid petitioner |
| 5 | Meeting exception | If didn't meet recently |
| 6 | Relationship duration | If engaged + valid petitioner |
| 7 | Free to marry | If engaged + valid petitioner |
| 8 | Income level | If engaged + valid petitioner |
| 9 | Joint sponsor | If income below threshold |
| 10 | Criminal history | If engaged + valid petitioner |
| 11 | Evidence of relationship | If engaged + valid petitioner |

## Scoring Logic

Located in `frontend/src/lib/scoring.js` → `calculateK1Score()`

### Hard Requirements (must pass all)
1. **Engaged** - Must select "We are engaged to be married"
2. **US Citizen Petitioner** - Quiz taker is US citizen OR partner is US citizen
3. **In-Person Meeting** - Met within 2 years OR qualifies for waiver
4. **Free to Marry** - Both parties legally single/divorced/widowed

### Soft Factors (affect likelihood rating)
- Income level (125% poverty guideline threshold)
- Joint sponsor availability
- Relationship duration
- Criminal history
- Evidence strength

### Likelihood Ratings
| Score | Rating |
|-------|--------|
| ≥85 | High |
| ≥65 | Medium |
| ≥45 | Low |
| <45 | Unlikely |
| Failed hard req | Ineligible |

## Dynamic Question Text

Questions can have different wording based on who's taking the quiz:

```javascript
{
  id: 'k1_income_level',
  text: "What is the US citizen's annual household income?",
  textByRole: {
    us_citizen: "What is your annual household income?",
    foreign_national: "What is your fiancé(e)'s annual household income?"
  }
}
```

The `getQuestionText(question, answers)` function resolves the correct text.

## Document Requirements

The database seeds 16 document requirements for K-1:

**Identity:**
- Form I-129F
- Proof of US Citizenship
- Passport Photos
- Beneficiary Passport
- Birth Certificate
- Divorce Decree (if applicable)

**Financial:**
- Form I-134 (Affidavit of Support)
- Petitioner Tax Returns
- Employment Letter
- Pay Stubs

**Supporting:**
- Proof of In-Person Meeting
- Proof of Relationship
- Engagement Evidence
- Police Certificates
- Medical Exam Results

## K-1 DIY Dashboard

After completing the quiz, authenticated users access the DIY dashboard at `/k1/dashboard`.

### Dashboard Architecture

The dashboard uses a **sidebar-based layout** with phase navigation:

| Component | Path | Purpose |
|-----------|------|---------|
| `K1Sidebar` | `components/dashboard/K1Sidebar.jsx` | Phase navigation with progress indicators |
| `K1MainContent` | `components/dashboard/K1MainContent.jsx` | Document lists by phase, overview with priority items |
| `DocumentCard` | `components/dashboard/DocumentCard.jsx` | Single-CTA cards (Start → Complete) |
| `DocumentPanel` | `components/dashboard/DocumentPanel.jsx` | Slide-out detail panel with video support |
| `CommentThread` | `components/dashboard/CommentThread.jsx` | Per-document notes/comments |
| `VideoModal` | `components/dashboard/VideoModal.jsx` | Full-screen video playback |
| `K1DashboardLayoutNew` | `components/dashboard/K1DashboardLayoutNew.jsx` | Layout orchestrating all panels |

### UX Patterns

- **Phase-based navigation** - Documents grouped by K-1 timeline phases (I-129F Filing, NVC, Embassy Interview, etc.)
- **Non-linear progress** - Users can work on any document, not locked to order
- **Priority indicators** - Bottleneck documents (police certs, medical exams) highlighted with "Priority" badges
- **Single CTA per document** - "Start" or "Complete" button, not 4 status buttons
- **Slide-out panels** - Details and comments in panels, not inline accordions
- **Video support** - Instructional videos show as thumbnails in panel, open in modal

### Adding Instructional Videos

Add a `video` object to any document in `frontend/src/data/k1Guidance.js`:

```javascript
'Form I-129F': {
  description: '...',
  video: {
    title: 'How to Fill Out Form I-129F',
    description: 'Jake walks through each section',
    youtubeId: 'abc123',  // or vimeoId, or url for direct video
    duration: '12:34',
    presenter: 'Jake'
  },
  // ... rest of guidance
}
```

### Dashboard API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/k1/dashboard` | GET | Dashboard summary (stats, latest assessment) |
| `/k1/documents` | GET | All K-1 documents with user progress |
| `/k1/documents/:id/progress` | PUT | Update document status |
| `/k1/documents/:id/comments` | GET | Fetch comments for a document |
| `/k1/documents/:id/comments` | POST | Add a comment |

| Table | Purpose |
|-------|---------|
| `document_requirements` | K-1 document definitions |
| `user_document_progress` | User's status per document |

Run migration `004_document_comments.sql` to add the comments table.
